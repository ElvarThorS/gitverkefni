2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone
	git log
	git status
	git diff
	git checkout
2. git clone = Clone a repository into a new directory
   git log = Show commit logs
   git status = Show the working tree status
   git diff = Show changes between commits, commit and working tree, etc
   git checkout =  Switch branches or restore working tree files


3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff 
	b) git diff --staged 
	c) git diff commit1 commit2
3. a) Show changes between commits, commit and working tree, etc
   b) will only show changes to files in the "staged" area
   c) eitthvað krakka shit

7.Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT?
7. Working directory = Directoryið sem þú ert að vinna í.
   Staging area = Staging is a step before the commit process in git. 
   Repository = a digital directory or storage space where you can access your project


8.Hvenær er sniðugt að nota greinar (branches)? 
8. A branch in Git is simply a lightweight movable pointer 
   to one of these commits. The default branch name in Git is master.
   As you initially make commits, you're given a master branch that 
   points to the last commit you made. Every time you commit, it moves 
   forward automatically.
